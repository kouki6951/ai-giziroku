# 議事録支援Webアプリ 設計書

## 1. 目的・背景

商談・面談・会議における会話を、AmiVoice API によりリアルタイムに音声認識し、自分／相手の発言を分離して画面上に表示する。さらに任意のタイミングで会話履歴を Claude API に送信し、追加ヒアリング事項・確認すべき論点・リスクなどを提案させ、会議終了後には議事録を自動生成する。生成された議事録および Claude 提案履歴は DB に保存し、後から検索・閲覧できる。

本フェーズでは認証機構を持たないクローズドな社内利用を想定し、最小機能で動作する MVP を作る。

## 2. スコープ

### In Scope
- ブラウザマイクからの音声取得とリアルタイム文字起こし
- 自分／相手の発言識別（マイク入力ソースで区別）
- 会議の内容（メタ情報）とフィードバック対象ロールの指定
- Claude API による議事中ヒアリング提案（選択ロール別／未選択時はロール非依存）
- 会議終了時の Claude API による議事録生成（バックグラウンド実行）
- 議事録・提案履歴の保存と閲覧

### Out of Scope（次フェーズ以降）
- 認証・ユーザー管理・組織管理
- 話者ダイアライゼーション（同一音声ストリーム内での話者分離）
- リアルタイム翻訳
- 録音音声ファイルの保存（ストレージ要件外）
- 外部カレンダー連携・Slack 通知

## 3. システム構成

```
┌────────────────────────────────────────────────────────────────┐
│                          Browser                                │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ Next.js (App Router) Client                              │  │
│  │                                                          │  │
│  │  Mic (self)                  Loopback (partner)          │  │
│  │   getUserMedia                getDisplayMedia            │  │
│  │   ↓                            ↓                         │  │
│  │   AudioWorklet (PCM 16kHz/16bit LE)                      │  │
│  │   ↓                            ↓                         │  │
│  │   WebSocket A ───────┐   ┌──── WebSocket B                │  │
│  │                      │   │                               │  │
│  │   ※AmiVoice 接続を 2 並列で張る (同一 APP_KEY)            │  │
│  └────────────┬─────────┴───┴──────────────────────────────┘  │
└───────────────┼────────────────────────────────────────────────┘
                │ (確定文の保存・Claude 呼び出し)
                ▼
        ┌──────────────────────┐    ┌────────────────────┐
        │ Next.js Route Handler│    │ AmiVoice Cloud API │
        │  - /api/meetings     │    │ (wss × 2 sessions) │
        │  - /api/transcripts  │    └────────────────────┘
        │  - /api/claude/*     │
        └────────┬─────────────┘
                 │
       ┌─────────┴─────────┐
       ▼                   ▼
┌──────────────┐    ┌──────────────┐
│ Claude API   │    │ SQLite       │
│ (Anthropic)  │    │ (Prisma)     │
└──────────────┘    └──────────────┘
```

- フロントエンド: Next.js 15 (App Router) + React 19 + TypeScript
- スタイリング: Tailwind CSS + shadcn/ui
- ORM: Prisma
- DB: SQLite（開発）/ PostgreSQL（本番に切替可能）
- 状態管理: Zustand（議事中の発言バッファ・接続状態）
- データ取得: TanStack Query（一覧・詳細）
- 音声: Web Audio API + AudioWorklet → AmiVoice WebSocket
- LLM: Anthropic Claude API（`claude-sonnet-4-6` 想定）

## 4. 機能一覧

| #   | 機能                  | 概要                                                |
| --- | --------------------- | --------------------------------------------------- |
| F01 | ミーティング開始      | タイトル・会議の内容（メタ情報）入力＋フィードバック対象ロール選択 → meeting レコード作成 → 作成画面へ |
| F02 | リアルタイム音声認識  | マイク(self)＋ループバック(partner)を 2 系統並列で AmiVoice 送信、確定文を表示 |
| F03 | 発言者別表示          | speaker_type ごとに発言ログをタイムライン表示       |
| F04 | 発言保存              | 確定発言を逐次 transcripts に保存                   |
| F05 | Claude 提案生成       | ボタン押下で直近会話を Claude に送信、選択ロール別（未選択時はロール非依存）の提案を表示。ロール別はタブで切替 |
| F06 | 提案履歴保存          | 提案結果を claude_feedbacks に保存                  |
| F07 | 議事録生成            | 「議事録を生成して終了」ボタンで会議を終了し、Claude による要約・整形を `after()` でバックグラウンド実行。生成中は `summary_status=processing` |
| F08 | 議事録保存            | 生成完了で meeting_summaries に保存し `summary_status=done`（失敗時 `error`）、ended_at を更新。詳細画面はポーリングで自動反映 |
| F09 | 過去議事録一覧        | 全 meeting を新しい順に表示                         |
| F10 | 過去議事録詳細        | 発言ログ・議事録・提案履歴を時系列で閲覧            |

## 5. 非機能要件

| 項目             | 要件                                                                 |
| ---------------- | -------------------------------------------------------------------- |
| 想定同時会議数   | 1 ブラウザ 1 会議。社内で同時 5 会議程度                             |
| 文字起こし遅延   | 発言終端から確定文表示まで 2 秒以内（AmiVoice 仕様準拠）             |
| 提案生成遅延     | ボタン押下から表示まで 10 秒以内                                     |
| ブラウザ         | Chrome / Edge 最新版（AudioWorklet と getUserMedia をサポート）      |
| 機密             | 会話本文を平文で DB 保存。社内 LAN 内運用、HTTPS 必須                |
| ログ             | サーバアクセスログとエラーログ。会話内容のログ出力は禁止             |
| 復旧             | DB は日次バックアップ。会議中の WebSocket 切断は自動再接続           |

## 7. 外部 API

### 7.1 AmiVoice Cloud Platform（リアルタイム音声認識）
- 認証: `Authorization` ヘッダの APP_KEY
- 接続: WebSocket Secure（wss）。詳細は別紙「音声ストリーム取得方式」参照
- エンジン: 汎用会話モデル（`-a-general` 等）

### 7.2 Anthropic Claude API
- SDK: `@anthropic-ai/sdk`
- モデル: `claude-sonnet-4-6`（既定）
- 用途: 議事中ヒアリング提案・議事録生成
- システムプロンプト: `lib/roles.ts` で会議の選択ロール（`roles`）・メタ情報（`description`）に応じて動的に組み立てる。ロール選択時はロール別の視点、未選択時はロール非依存の提案／議事録になる
- プロンプトキャッシュ: **システムプロンプトのみ** `cache_control: { type: "ephemeral" }` を付与する
  - 会話履歴側にはキャッシュを設定しない（毎回内容が伸びるため、TTL 維持コストに見合わない）
  - 提案用と要約用でシステムプロンプトを分け、それぞれ独立にキャッシュさせる
- 出力形式: いずれも Markdown 文字列。DB へもそのまま Markdown で保存（パースしない）

## 8. セキュリティ／プライバシ

- 認証なしのため、社内 LAN または VPN 配下での運用に限定する
- AmiVoice APP_KEY と Anthropic API キーはサーバサイドの環境変数のみで扱い、ブラウザに露出させない
- AmiVoice への音声送信は WebSocket をブラウザから直接張る運用とするが、APP_KEY はサーバ側で発行する短期トークン経由で渡す（Route Handler 経由で取得）
- ブラウザコンソール・サーバログに発話本文を出力しない

## 9. 想定リスクと対応

| リスク                                            | 対応                                                                          |
| ------------------------------------------------- | ----------------------------------------------------------------------------- |
| 画面共有で「音声を共有」が OFF にされる           | `getAudioTracks().length === 0` を検知してトースト案内。`self` のみで継続可    |
| 「ウィンドウ」共有を選ばれる                      | Chrome 仕様で音声不可。トーストで「タブ または 画面全体 を選択」と案内         |
| macOS デスクトップアプリ（Slack/Zoom）の音声      | Chrome が macOS のシステム音声を拾えない。Web 版誘導 or 仮想オーディオデバイス（Phase 7 で対応） |
| ループバック音声に自分の声が混入する              | partner 側は混入を許容（重複扱い）。エコーキャンセル相当は次フェーズで検討     |
| 相手音声がスピーカから流れ、self マイクに回り込む  | partner ON 中はヘッドホン推奨バナーを常時表示                                  |
| AmiVoice 同時 2 セッションの契約上限              | 契約プラン側で 2 並列が許容されているかを事前確認。超過時はエラーを画面提示    |
| WebSocket 切断（2 系統のいずれか）                | 該当セッションのみ自動再接続。もう一方は止めない。直前の確定文以降は再送しない |
| Claude のレスポンス遅延                           | UI 上にローディング表示。重複送信を防止                                       |
| 長時間会議での文字数肥大                          | Claude 送信時は直近 N 件 + 古い発言の要約（運用で N をチューニング）         |

## 10. 用語

| 用語         | 意味                                                       |
| ------------ | ---------------------------------------------------------- |
| 確定文       | AmiVoice が `A` イベント等で返す最終結果テキスト           |
| 中間結果     | `U` イベント等で返す未確定テキスト。表示のみ、DB 保存しない |
| speaker_type | 発言者種別。`self`（自分＝マイク） / `partner`（相手＝ループバック） |
| ループバック  | ブラウザ／OS で再生中の音声を `getDisplayMedia({audio:true})` で取り込むこと |
