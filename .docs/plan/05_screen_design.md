# 画面設計書

## 1. 画面一覧

| ID  | 画面名               | パス                          | 概要                                |
| --- | -------------------- | ----------------------------- | ----------------------------------- |
| S01 | 議事録一覧画面       | `/`                           | 過去の議事録カードを新しい順に表示  |
| S02 | 新規ミーティング作成 | `/new`                        | タイトル入力 → 録音画面へ           |
| S03 | 議事録作成（録音）   | `/[id]/recording`             | リアルタイム発言ログ＋Claude 提案    |
| S04 | 議事録詳細画面       | `/[id]`                       | 完成した議事録・発言ログ・提案履歴   |

## 2. 共通レイアウト

- ヘッダ: アプリ名 `AI-Giziroku`、右に「新規ミーティング開始」ボタン
- フッタ: なし
- カラーパレット: 水色（sky）×ライム（lime）を基調。ボタンはピル型などを採用
- フォント: 日本語は `Inter` + `Noto Sans JP` のフォールバック
- 録音中は赤丸インジケータ＋経過時間を画面上部に常時表示

## 3. S01: 議事録一覧画面

### 目的
過去の議事録を素早く見つけて開く。新規ミーティングを開始する。

### ワイヤーフレーム（PC）
```
┌────────────────────────────────────────────────────────────────────┐
│ AI-Giziroku                                    [＋ 新規ミーティング]│
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  議事録一覧                                                        │
│                                                                    │
│  ┌──────────────────────────┐ ┌──────────────────────────┐         │
│  │ 顧客A 商談 #1            │ │ 顧客B 定例               │         │
│  │ 2026/05/28 14:00 – 15:02 │ │ 2026/05/27 10:00 – 10:35 │         │
│  │ 課題: 3 / TODO: 5        │ │ 課題: 1 / TODO: 2        │         │
│  └──────────────────────────┘ └──────────────────────────┘         │
│  ┌──────────────────────────┐                                      │
│  │ 採用面接                 │                                      │
│  │ 2026/05/26 11:00 – 11:48 │                                      │
│  └──────────────────────────┘                                      │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

### 構成要素
- **カードリスト**: `meetings` の `title`, `started_at`, `ended_at`, 発言件数・議事録件数。議事録生成中（`summary_status=processing`）のカードには「議事録を生成中」バッジを表示
- **新規ミーティングボタン**: `/new` へ遷移
- **空状態**: 「まだ議事録がありません。新規ミーティングを開始してください。」

### 操作
| 操作                 | 挙動                          |
| -------------------- | ----------------------------- |
| カードクリック       | `/[id]` へ遷移                |
| 新規ミーティング     | `/new` へ遷移                 |

## 4. S02: 新規ミーティング作成画面

### 目的
タイトル・会議の内容（メタ情報）を入力し、フィードバック対象ロールを選んで録音画面に入る。

### ワイヤーフレーム
```
┌─────────────────────────────────────────────┐
│ 新しいミーティング                          │
├─────────────────────────────────────────────┤
│ タイトル                                    │
│ ┌─────────────────────────────────────────┐ │
│ │ 顧客A 商談 #2                           │ │
│ └─────────────────────────────────────────┘ │
│ 会議の内容・目的（メタ情報）                │
│ ┌─────────────────────────────────────────┐ │
│ │ 目的・議題・前提となる背景 …            │ │
│ └─────────────────────────────────────────┘ │
│ フィードバックを求めるロール                │
│ [✓営業] [✓ﾏｰｹ] [✓ｴﾝｼﾞﾆｱ] [経営] [ﾃﾞｻﾞｲﾅｰ] [PM]│
│                                             │
│         [キャンセル]   [開始]               │
└─────────────────────────────────────────────┘
```

### 構成要素
- タイトル入力（必須、200 字以内）
- 会議の内容・目的（メタ情報）入力（任意）。フィードバック／議事録のコンテキストとして利用
- フィードバックを求めるロール選択（複数選択可・未選択可）。ロール: 営業 / マーケティング / エンジニア / 経営・マネジメント / デザイナー / プロダクトマネージャー。未選択時はロール非依存のフィードバックになる
- 「開始」ボタン: `POST /api/meetings`（title・description・roles）→ 返却 ID で `/[id]/recording` へ
- 「キャンセル」: `/` に戻る

### バリデーション
- 空欄禁止
- 先頭末尾空白は trim

## 5. S03: 議事録作成（録音）画面

### 目的
録音中の発言をリアルタイム表示し、随時 Claude に提案を依頼。終了時に議事録を生成して保存。

### ワイヤーフレーム（PC: 2 カラム）
```
┌──────────────────────────────────────────────────────────────────────┐
│ ● 録音中 00:12:34   顧客A 商談 #2      mic●self  loop○partner       │
├──────────────────────────────────┬───────────────────────────────────┤
│ 発言ログ                         │ Claude 提案                       │
├──────────────────────────────────┼───────────────────────────────────┤
│ 14:00:01 自分                    │ 確認不足                          │
│  本日はよろしくお願いします       │  ・予算                           │
│                                  │  ・導入時期                       │
│ 14:00:08 相手                    │  ・決裁者                         │
│  よろしくお願いします            │                                   │
│                                  │ 次に聞くべき質問                  │
│ 14:00:15 自分                    │  - 現行運用での課題は？           │
│  まず御社の現在の状況を…         │                                   │
│                                  │ リスク                            │
│ … (auto scroll)                  │  - 決裁ルート未確認               │
├──────────────────────────────────┴───────────────────────────────────┤
│ [相手音声を取り込む] [フィードバック取得] [議事録を生成して終了] [終了のみ]│
└──────────────────────────────────────────────────────────────────────┘
```
※ ヘッダ右上は 2 系統の接続インジケータ。`mic●` = マイク接続中、`loop○` = ループバック未接続。`speaker_type` は系統で自動決定するため、UI トグルは持たない。

### 構成要素
- **録音インジケータ**: 経過時間 + 2 系統（mic / loop）の接続状態（接続中／再接続中／停止）
- **ヘッドホン推奨バナー**: partner 系統が ON のとき画面上部に常時 1 行表示「相手の声がスピーカから流れていると重複認識されます。ヘッドホンを使用してください」
- **発言ログペイン**: 確定文を時系列で。`self` / `partner` をラベル＋色で識別。中間結果は最終行に薄字で重ねる
- **Claude 提案ペイン**: 取得した最新フィードバック（Markdown）。選択ロール別の場合はロールごとのタブで切替表示。下に折りたたみ式の履歴
- **ボタン群**:
  - 「相手音声を取り込む」 `getDisplayMedia` を呼んで partner 系統を起動。すでに ON ならラベルが「相手音声を停止」に変わる
    - 押下前に「対応会議ツール案内」のポップオーバーを 1 回だけ表示（Web 会議は タブ共有、Slack デスクトップは 画面全体共有 を選ぶ等）
  - 「フィードバック取得」 `POST /api/claude/feedback`
  - 「議事録を生成して終了」 `POST /api/meetings/[id]/end`（ended_at 確定）→ `POST /api/claude/summary`（議事録生成をバックグラウンド開始）→ 生成完了を待たず一覧画面へ即遷移
  - 「終了のみ」 `POST /api/meetings/[id]/end`（議事録を生成せず会議だけ終了し詳細画面へ）

### 状態遷移（同画面内）
```
idle ─[開始]→ connecting(self) ─→ recording(self only)
                                       │
                                       │ [相手音声を取り込む]
                                       ▼
                                  connecting(partner) ─→ recording(self+partner)
                                                              │
                              ┌─[フィードバック取得]─────────┘
                              ▼
                       fetching_feedback ─→ recording
                              │
                              │ [議事録を生成して終了]
                              ▼
                       summary_processing ─→ 一覧へ即遷移（生成はバックグラウンド）
                              ▲
                              │ [切断検知/再接続] (系統ごとに独立)
                              └ reconnecting(self|partner)
```

### 入力検証・ガード
- 録音中の画面離脱は `beforeunload` で警告
- 「議事録生成」中の二重押下を防ぐローディング
- マイク権限拒否時はオーバーレイで権限の許可方法を案内

## 6. S04: 議事録詳細画面

### 目的
完成した議事録、会議の内容（メタ情報）、発言ログ、Claude 提案履歴を一画面で見直す。

### ワイヤーフレーム
```
┌────────────────────────────────────────────────────────┐
│ 顧客A 商談 #2     2026/05/28 14:00 – 15:02             │
├────────────────────────────────────────────────────────┤
│ 会議の内容・目的（メタ情報があれば表示）               │
├────────────────────────────────────────────────────────┤
│ 議事録                                                 │
│ ## 会議概要                                            │
│ ## 課題                                                │
│ ## 要望                                                │
│ ## 決定事項                                            │
│ ## TODO                                                │
├────────────────────────────────────────────────────────┤
│ 発言ログ（折りたたみ）                                 │
│ 14:00:01 自分: 本日はよろしくお願いします              │
│ 14:00:08 相手: よろしくお願いします                    │
│ …                                                      │
├────────────────────────────────────────────────────────┤
│ Claude 提案履歴                                        │
│ 14:25:00 取得 ── 確認不足: 予算 / 導入時期 …           │
│ 14:50:00 取得 ── 次の質問: 競合検討状況 …              │
└────────────────────────────────────────────────────────┘
```

### 構成要素
- ヘッダ: タイトル、開始終了時刻
- 会議の内容・目的セクション: メタ情報（`description`）があれば表示
- 議事録セクション: Markdown レンダラ（`react-markdown` 等）。生成中（`summary_status=processing`）は「生成中」表示＋ポーリング（`summary-status-poller.tsx`）で完了を自動反映。失敗（`error`）時はエラー表示
- 発言ログセクション: 折りたたみ、初期は閉じる
- 提案履歴セクション: 取得順に縦並び。ロール別はロールごとのタブで切替表示（`role-feedback-tabs.tsx`）

### 操作
| 操作                       | 挙動                                    |
| -------------------------- | --------------------------------------- |
| 「議事録をコピー」         | クリップボードへコピー                  |
| 「一覧へ戻る」             | `/` へ遷移                              |
| 「録音画面へ」（進行中のみ） | `/[id]/recording` へ遷移                |
| 「削除」                   | `DELETE /api/meetings/[id]`（関連も CASCADE 削除） |

## 7. レスポンシブ設計

| ブレークポイント | 想定端末       | レイアウト                                                                 |
| ---------------- | -------------- | -------------------------------------------------------------------------- |
| `>= 1024px`      | PC             | 録音画面は 6:4 のままの 2 カラム                                            |
| `768px – 1023px` | タブレット     | 録音画面は 6:4 だがペイン内のフォントを 1 段階小さく                       |
| `< 768px`        | スマートフォン | 録音画面は上下 2 段。上 = 発言ログ（高さ 60vh）／下 = Claude 提案（40vh） |

スマホ時の録音画面ワイヤー:
```
┌────────────────────┐
│ ● 00:12:34         │
│ mic●self loop○peer │
├────────────────────┤
│ 発言ログ           │
│ …                  │
│ …                  │
├────────────────────┤
│ Claude 提案        │
│ …                  │
├────────────────────┤
│ [相手取込][取得]    │
│ [生成して終了][終了のみ]│
└────────────────────┘
```

## 8. アクセシビリティ

- すべての操作ボタンに `aria-label` を付与
- 録音インジケータは色＋アイコン＋テキストで状態を表現
- 自動スクロールは「自分が手動でスクロール中はオフ」にして読み戻しを邪魔しない
- キーボード操作で「フィードバック取得」「議事録生成」を発火できるショートカット（`Ctrl+Enter` 等）

## 9. エラーメッセージ

| シナリオ                  | 表示                                              |
| ------------------------- | ------------------------------------------------- |
| マイク権限拒否            | オーバーレイ「マイクを許可してください」          |
| AmiVoice 接続失敗         | トースト「音声認識サーバに接続できません。再試行」 |
| 画面共有で音声が共有されない | トースト「相手音声が共有されていません。『音声を共有』をONにして再選択」＋「対応会議ツール一覧」へのリンク |
| 「ウィンドウ」を共有された   | トースト「ウィンドウ共有では音声を取得できません。**Chrome タブ** または **画面全体** を選択してください」 |
| macOS で画面全体音声が来ない | トースト「macOS ではデスクトップアプリの音声を直接取得できません。Web 版会議を使うか、仮想オーディオデバイスの設定をご確認ください」＋ヘルプ動線 |
| 画面共有を途中で停止       | partner 系統のみ停止。「相手音声を取り込む」ボタンで再開可能            |
| Claude API エラー         | トースト「提案の取得に失敗しました」              |
| 議事録生成失敗            | トースト「議事録の生成に失敗しました」            |
| ネットワーク切断（一般）  | バナー「オフラインです」                          |
